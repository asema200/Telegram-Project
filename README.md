Система осуществляет сбор, анализ и визуализацию данных о вакансиях с использованием API HeadHunter, предоставляя пользователям статистическую информацию о состоянии рынка труда по различным профессиям.

## 4. Реализация

### 4.1. Модуль сбора данных (stats_advanced.py)

Класс `VacancyStats` реализует паттерн **Strategy** для гибкой обработки различных типов запросов.

**Алгоритм сбора данных:**
1. Формирование HTTP-запроса с параметрами фильтрации
2. Пагинация результатов (до 2000 вакансий)
3. Нормализация JSON-структуры в DataFrame
4. Применение задержек (0.5с) для соблюдения rate limiting

**Временная сложность:** O(n), где n - количество вакансий
**Пространственная сложность:** O(n)

### 4.2. Модуль статистического анализа

**Реализованные метрики:**

1. **Медиана** (Median)
   - Наиболее устойчивая к выбросам метрика
   - Формула: x̃ = (x[n/2] + x[n/2+1]) / 2 для четного n

2. **Среднее арифметическое** (Mean)
   - μ = (Σxi) / n

3. **Перцентили** (25%, 75%)
   - Характеризуют квартильное распределение

4. **Стандартное отклонение** (Standard Deviation)
   - σ = √(Σ(xi - μ)² / n)

**Обработка выбросов:**
- Фильтрация значений > 1,000,000 ₽
- Исключение нулевых значений
- Винсоризация экстремальных значений

### 4.3. Модуль визуализации

Реализованы следующие типы графиков:

**1. Гистограмма распределения зарплат**
- Количество bins: 30 (по правилу Стерджеса)
- Наложение линий медианы и среднего
- Формат: PNG, 300 DPI

**2. Сравнительная диаграмма профессий**
- Столбчатая диаграмма (bar chart)
- Параллельное отображение медианы и среднего
- До 4 профессий на одном графике

**3. Коробчатая диаграмма (Box plot)**
- Визуализация квартилей
- Идентификация выбросов
- Компактное представление статистики

### 4.4. Алгоритм профориентационного тестирования

**Методология:**

Разработан упрощенный психометрический тест, основанный на теории профессиональных интересов Голланда, адаптированный для цифрового формата.

**Структура теста:**
- Количество вопросов: 7
- Варианты ответа: 4 (на каждый вопрос)
- Оцениваемые черты: 10 (technical, analytical, creative, social и др.)

**Алгоритм обработки результатов:**

1. Подсчет баллов по каждой черте характера
2. Агрегация в профессиональные профили:
   - Техническая сфера
   - Аналитика и данные
   - Креативная сфера
   - Работа с людьми

3. Определение доминирующего профиля по максимальной сумме баллов
4. Генерация персонализированных рекомендаций

**Профили результатов:**

| Профиль | Доминирующие черты | Рекомендуемые профессии |
|---------|-------------------|-------------------------|
| Технический | technical, logical | Программист, DevOps, Системный администратор |

## 6. Математическое обоснование

### 6.1. Статистические методы

**Выбор медианы как основной метрики обоснован следующим:**

Медиана обладает робастностью к выбросам, что критично для данных о заработных платах, где присутствуют экстремальные значения (топ-менеджмент, специалисты редких профессий).

**Сравнение метрик:**

Для выборки зарплат [50k, 60k, 70k, 80k, 500k]:
- Среднее: 152k (искажено выбросом)
- Медиана: 70k (устойчива к выбросу)

### 6.2. Обработка данных

**Конвертация валют:**
```
RUB = {
    USD × 80,  если валюта = USD
    EUR × 75,  если валюта = EUR
    salary,    если валюта = RUB
}
```

**Расчет заработной платы "на руки":**
```
NET = {
    GROSS × 0.87,  если gross = true
    salary,        если gross = false
}
```
где 0.87 = 1 - 0.13 (НДФЛ)

### 6.3. Алгоритм классификации профилей

**Функция подсчета баллов:**

```
Score(profile) = Σ(trait_count[trait]) для всех trait ∈ profile_traits
```

**Функция классификации:**

```
Result = argmax(Score(profile)) для profile ∈ Profiles
```

где Profiles = {technical, analytical, creative, social, balanced}
